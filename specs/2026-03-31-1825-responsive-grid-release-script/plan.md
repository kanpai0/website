# Session — 2026-03-31 — Responsive grid & release script

## What was built

### 1. Responsive recipe grid (no breakpoints)
**File:** `static/css/main.css` — `.recipes-grid`

Replaced a fixed `grid-template-columns: 1fr 1fr` with:
```css
grid-template-columns: repeat(auto-fill, minmax(clamp(180px, 25% - 6px, 300px), 1fr));
```

- `clamp(180px, 25% - 6px, 300px)` gives fluid column sizing without breakpoints
- `180px` floor → 2 columns on phone (~375px)
- `25%` preferred → ~4 columns on tablet (~768px)
- `300px` ceiling → 4–5 columns on desktop (1440px+)
- Trade-off accepted: column count is approximate, not pixel-perfect at every viewport

**Why not breakpoints:** user preference for a breakpoint-free approach. Acknowledged the clamp trick is an approximation, not an exact 2→3→4 step function.

### 2. Release script fixes (`scripts/release.sh`)

#### a. `mapfile` compatibility
**Problem:** `mapfile` is bash 4+; macOS ships bash 3.2 → `command not found` at runtime.
**Fix:** replaced with a `while IFS= read -r line` loop, which works on bash 3.2.

#### b. Changelog separator bug
**Problem:** `git-cliff --prepend` produced `---## [1.0.0]` (no blank lines between separator and next release header).
**Fix:** switched from `--prepend` to manual concatenation:
```bash
git-cliff ... > tmp
printf '\n---\n\n' >> tmp
cat CHANGELOG.md >> tmp
mv tmp CHANGELOG.md
```
This guarantees `\n---\n\n` between releases.

#### c. No auto-push
**Problem:** script was pushing to `origin main` and the tag automatically.
**Fix:** removed both `git push` calls. Script now only commits and tags locally. A reminder is printed at the end with the manual push command.

#### d. Removed auto-generated `specs/release-notes.md`
**Problem:** release script was writing a `specs/<date>-release-vX.Y.Z/release-notes.md` on every release, cluttering the specs directory with auto-generated content.
**Decision:** specs are authored by Claude during work sessions (like this file), not auto-generated by the release script.

#### e. Backlog → Réalisées promotion
**Problem:** the interactive backlog step was marking items `[x]` in place, not moving them to the `## Fonctionnalités — Réalisées ✅` section.
**Fix:** rewrote step 8 to:
1. Collect picked items and their line numbers
2. Find the `---` separator just before `## Fonctionnalités — En cours`
3. Rewrite the README in a single pass: insert moved items before that separator, skip the original lines

Items are moved with their full text preserved (e.g. `**Grille responsive** : colonnes fluides…`), stripped of the `- [ ] ` prefix.
